Banco de Dados

<p>Pode-se definir que um banco de dados é qualquer coleção de informações e dados inter-relacionados, ou seja, sempre que for possível agrupar informações que se relacionam e tratam de um mesmo assunto, pode-se dizer que tenho um banco de dados.<br>
O banco de dados pode ser de diversas formas, como por exemplo fazer uma lista de compras em um pedaço de papel, estoque de um mercado, uma lista telefônica, ou um sistema de controle de RH de uma empresa.<br>
Um sistema de gerenciamento de banco de dados (SGBD) é um software que possui recursos capazes de manipular as informações do banco de dados e interagir com o usuário. E é estruturado com um conjunto de quatro componentes básicos: 
<ul>
<li>dados</li>
<li>hardware,/li>
<li>software e usuários.</li>
</ul>

Exemplos de SGBDs são: Oracle, SQL Server, DB2, PostgreSQL, MySQL, o próprio Access ou Paradox, entre outros.<br>
Os bancos de dados são usados para armazenar e organizar dados, de modo que seja mais fácil gerenciá-los e acessá-los. Um dos objetivos são o de isolar o usuário dos detalhes internos do banco de dados (promover a abstração de dados) e promover a independência dos dados em relação às aplicações, ou seja, tornar independente da aplicação, a estratégia de acesso e a forma de armazenamento.<br>


O que são dados?
Dados referem-se a qualquer informação capturada e armazenada sobre uma pessoa, um lugar, uma coisa ou um objeto, chamada de entidade, bem como os atributos dessa entidade.


<h2>Tipos de bancos de dados</h2>
<p>Os tipos de bancos de dados são amplamente agrupados em bancos de dados relacionais e não relacionais. Bancos de dados relacionais são altamente estruturados e compreendem uma linguagem de programação chamada SQL (Structured Query Language). Os bancos de dados não relacionais são altamente diversificados, dando suporte a uma variedade de estruturas de dados. Como muitos bancos de dados não relacionais não usam o SQL, eles costumam ser chamados de bancos de dados NoSQL.</p><br>

<h2>Tipos de estruturas de dados</h2>
As estruturas de tabela são estruturas de banco de dados relacionais que organizam dados em linhas e colunas (as linhas contêm entidades, enquanto as colunas contêm atributos da entidade). As tabelas largas (ou repositórios de colunas largas) usam colunas esparsas com atributos vazios para aumentar consideravelmente o número total de colunas que você pode ter na tabela. Como alguns espaços ficam vazios, as tabelas largas são um exemplo de uma estrutura de banco de dados não relacional.
